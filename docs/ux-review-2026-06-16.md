# ericcaskey.com — Holistic UX Review (FDS-24)

**Date:** 2026-06-16
**Reviewer:** frontend-design loop (FDS-24)
**Scope:** every ericcaskey.com surface — hero, about, work, writing, contact, 404, nav/footer — against the frontend-design lens and the `_shared/architecture/002` + `004` family contract.
**Method:** code read of `origin/main` plus first-hand build + screenshot QA of hero (FDS-2), contact (FDS-21 #68), nav toggle (FDS-22 #69), and 404 (FDS-23 #70) done in this session.

This is a findings doc. Concrete fixes are filed as child items (FDS-81, FDS-82); lower-severity observations are recorded with an explicit decision so they don't get re-litigated.

## What's already right (confirmed, no action)

- **Focus ring:** single `*:focus-visible` token ring, globally — consistent on every focusable element.
- **Dark-mode parity:** all surfaces theme through `--color-*` tokens; verified light/dark on contact, nav, 404.
- **Register split (001) intact:** arrivals-board tempo, dividers not cards, h1–h2 only, no card components — deliberately distinct from caskeycoding. Do **not** converge.
- **List-item hover:** `TimelineItem` and `ToolCard` use the correct border-led hover (title → accent, hairline → `--color-border-strong`, `transition-colors`, v2.1 §2.1).
- **Cross-domain handoff:** wordmark, theme-sync key (`eric-caskey-theme`), `?theme` handoff, reciprocal footer cross-link — all present (002).
- **In-flight this session:** #68 (contact field focus + status notes), #69 (toggle hover affordance), #70 (404 rebrand) already closed three affordance/branding gaps.

## Findings

| # | Sev | Surface | Finding | Fix |
|---|-----|---------|---------|-----|
| F1 | **High** | all pages | **h1 scale drift.** `globals.css` defines the canonical `h1 { clamp(2.5rem, 5.5vw, 4rem) }` (002 clash #8, the cross-domain handoff size). But it is overridden inconsistently per page: home hero = `clamp(2.5rem, 6vw, 4rem)` (6vw, not 5.5); about/work/writing/contact h1 = Tailwind `text-3xl md:text-4xl` (~1.875→2.25rem, **far smaller** than canonical); 404 = `clamp(2rem, 5vw, 3rem)`. Same `<h1>` element renders at **four different scales** across six surfaces, and only the unstyled global matches the contract. The word "About" on `/about` does not render at the size 002 promises. | **FDS-81** |
| F2 | **Med** | home, writing | **`EssayCard` lacks the border-led hover** its sibling list components have. `TimelineItem`/`ToolCard` step the hairline to `--color-border-strong` and the title to accent on hover with `transition-colors`; `EssayCard` (`components/EssayCard.tsx`) has none — only the inner `CrossSiteLink` gets the global link-hover. Inconsistent tactility between otherwise-identical divider rows (the same affordance-gap class as #68/#69). | **FDS-82** |
| F3 | Low | inner pages | **Staggered `.reveal` load is home-only** (and 404 via #70). about/work/writing/contact load with no motion. | Decision: **accept as a landing/error signature** — restraint on inner pages is on-voice (one orchestrated reveal per 004 §motion, not scattered). No child item; revisit only if the owner wants site-wide load motion. |
| F4 | Low | contact | **No eyebrow/locator** on `/contact` while about/work/writing/404 all open with an `.eyebrow`. | Decision: **leave as-is** — contact's h1 + one-line intro is already minimal; adding an eyebrow is redundant new copy. Noted, not filed. |
